Var vs Let vs Const

Important bits to understand

  • Declaration vs Initialization
    var d; = Declaration (creating a variable with undefinite)
    var d = "hello world"; = Initalization (Assigned a value after declaration)

  • Scope - Defines where variables and cuntions are accessible inside your program.
    Var = function scope, will not exist oustide of the function
    Let = block scope, available inside the block it created

  • Hoisting - The idea of declaring a variable without a assigning a value
    Var = undefined when trying to access before it's declaration
    Let = reference error when access before it's declaration

    In JS, (particularly functions), all var items used will be initially declared
    with undefinined in the start before it is used/assigned a value

  • Const = pretty much same as Let EXCEPT:
    Let can be reassigned , const can't
    It is not exactly immutable, you can reassign a property or change
    certain aspects of the const object however you cannot redeclare it

    • In summary:
    So to recap, var is function scoped and if you try to use a variable declared
    with var before the actual declaration, you’ll just get undefined. const and
    let are blocked scoped and if you try to use variable declared with let or
    const before the declaration you’ll get a ReferenceError. Finally the difference
    between let and const is that once you’ve assigned a value to const, you can’t
    reassign it, but with let, you can.
