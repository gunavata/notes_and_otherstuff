  2. CSS Type & Class Selectors

    3. Type selector i.e p, h1 etc

    Class selector i.e class="jumbotron" / .jumbotron

  3. CSS ID and Attribute Selectors

    4. ID selector i.e id="unique" / #unique

    Generally speaking, dont use ID

    Attribute selector i.e input name="name" / input[name]

  4. CSS DOM Selectors

    5. Direct Child selector i.e div then span / div > span

    Descendants of selector i.e div then span / div span

    Declare specific first, general later

    Multiple descendants of selector i.e div then span then ol / div span ol

    Sibling selector i.e div next to span / div ~ span

  5. CSS - Inline and Embedded

    6. Inline CSS - often used for debugging because fast, can slow dev process because needs to be constantly implemented

    i.e div style="border: 1px solid #4a4a4f;"

    Embedded CSS - in head element, not great but ok for styling that page

    i.e in head element, style element with style declrations inside targetting specific areas of the html doc

  6. External CSS and precedence

    7. External CSS - external css file linked to html via link element inside head

    i.e link rel="stylesheet" href="location of css.css" type="text/css"

    Best as can be minifed, recycled, faster loading and seperation of concerns

    Precendence of css styles = inline > embedded > external

    !important placed after value - forces that css to always be active/highest precedence (dont use this)
