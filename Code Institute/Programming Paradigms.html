What is it? - Way of thinking of software construction/methodology (Not language specific)

Types of PP

  • Declarative
    HTML - CSS - SQL
    Do something = WHAT HAPPENS?
    Avoid the nitty gritty of how stuff happens
    Ease of use!
    Limited to smaller instruction set

  • Imperative
    Java - C# (Sharp) - C++ - Javascript - Python - Ruby
    Do Something = HOW HAPPENS?
    Be explicit on how to do something
    Alot more control!
    Requires painstaking details
    Apply logic!

    Sequenced instructions (Algorithms) - Carry out instructions in order to do something
    Mutable data - Data that can change over the duration of a program, states can change
    Variables - Location where data is stored

  • Structured
    Extension of Imperative programming
    Make decision based on current states of data (If checks)
    Allows repetition of data (A loop)

  • Procedural
    Extension of Structured programming
    Creation of function (AKA procedures/subroutines)
    Good for breaking problems down into smaller chunks
    Write code once to be used again and again (Refactoring)

This is then pushed down to two major programming styles

Object Oriented Programming

See everything as OBJECTS with PROPERTIES and ACTIONS

Focus on grouping everything into classes, properties and behaviours that are common

Class is like a blueprint to create object

Objects are instances of the class

Event Driven Programming

Waits in the background for something to happen

Relies on triggers for codes to be put into action

All have role to play in order to create a software that elicits positive emotional response from users
