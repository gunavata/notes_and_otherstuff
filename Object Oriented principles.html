Object oriented principles

  • Simlarities between non OOP approach:
    Variables that store information and other data.
    Procedures (AKA functions, methods, routines) that take input, generate output and manipulate data

  • Objects and Clases:
    Usually support inheritence for code reusability and extensibility. Two support concepts
    Classes = data members and member functions grouped together (The blueprint)
    Objects = instance of the class (The created item from the blueprint)

  • Abstraction:
    A class that cannot be instantiated, it exist only to be inherited into other
    concrete classes.
    Employee abstracts into hourly vs salary followed by manager vs staff or fulltime vs parttime
    Shape abstract with methods to calculate display and area into rectangle, circle and triangle

  • Encapsulation:
    Binds together data and functions that manipulate it and keep it
    safe from outside interference and misues. Important part of data hiding
    It allows you to tamper with the attributes or access methods ONLY through
    the related methods in the class

  • Polymorphism:
    Concept where an object behaves differently in different situations.
    Core concept of method overloading where depending on what is passed/received
    the correct method is used i.e draw() vs draw(int) returns a different
    method due to what is passed as an argument

  • Inheritence - mechanism of basing an object or class upon another object or class
    retaining similiar implementation.
    Subclass(Child) & Superclass(Parent)

    Single inheritence = A -> B
    Multiple inheritence = A + B -> C
    Multilevel inheritence = A -> B -> C
    Hierarchical inheritence = A -> B && A -> C && A -> D
    Hybrid inheritence = Mix of 2 or more of the above

    Two important properties
    Overriding = the child class overrides the parent's class attribute or method
    to be more specific or unique
    Code reuse = Inheriting the parent class because most code is being used
    but with slight difference (hence overriding)

  • Association:
    Relationship between objects (Database relationship)
    One to many/Many to one/One to one/Many to many
    Both objects are independant of each other however

    Aggregation is a type of relationship that defines ownership
    Has-a type relationship.

    Composition is a type of aggregation that also specifies that
    the child cannot exist without the parent i.e a room object cannot exist
    without a house object
