NoSQL(MongoDB) vs SQL(MySQL)

Main difference is that
NOSQL databases do NOT use relational models i.e MongoDB
SQL Databases use relational models i.e MySQL

Relational database advantages (MySQL)
  • Less expensive since more mainstream
  • Better support, product suites, add ons (again due to more mainstream)
  • Atomicity of operations - database relies on entire operation completing otherwise not done at all
  • Must comply with integrity requirements for data type and compatibility
Relational database disadvantages (MySQL)
  • Atomicity operations affects performance
  • Scalability is usually lower than nosql
NoSQL (non relational) advantages (MongoDB)
  • Scalability and decentralized, supports distributed structures
  • More open and flexible as databases. Allow adapting to needs of a project much easier than relational models
  • Changes in schemas can be done without stoping database
  • Horizontal scaling, capable of growing in device number instead of residing in one large device
  • Can run on low resource devices
  • Query optimation on databases designed for large amounts of data
NoSQL (non relational) disadvantages (MongoDB)
  • Not all database contemplate atomicity of instructions and integrity of data. Expects eventual consistence.
  • Compatibility with SQL instructions. Have their own chracteristics in query language. Support for query in NoSQL is more complicated
  • Lack of standardizing unlike in relational databases. Uncertain future.
  • Cross platform support. Improvements being made but not as good as SQL.
  • Not many useful management tools or console access.
When to use SQL vs NoSQL
  • Data must be consistent with no room for error = SQL
  • Budget is low = NoSQL
  • Datastructures changes alot = NoSQL
  • Analyzing large quantities of data in read only = NoSQL
  • Event capture and processing = NoSQL
  • Online stores with complex intelligence engines = NoSQL
