Common Words and their definition

  • Mutable - Type of variable that can be changed.
    In JS, objects and arrays are mutable, not primitive values.

    Mutable - An object whose state can be modified after it is created.
    Immutables - objects whose state cannot be changed once it is created.
    Source

  • Data hiding - Allows us to change names of our instance variables
    without forcing the users of our object to change how they use it.
    Using a method to access instead of accessing the data fields

  • Map function - creates a copy of the previous array/item (therefore
    leaves the original dataset untouched)

  • Garbage collection - once the object/attribute/method (resource?) is no longer
    being referenced, then it can be cleared up to make space for memory

  • Public static void -

    Public means it is visible and can be called from other objects of other types

    Static means the method is associated with class, not an instance. It can be
    called without creating an object

    Void means it returns no value

  • Recurssion - the process of taking a big problem and sub dividing into
    multiple, smaller instances of the same problem
    In practice, its a function that calls itself and outputs a result
    once the stack has unwinded
