Object oriented programming vs Functional programming

OOP - programming paradigm based on concept of objects,
which are data structures that contain data
in the form of fields, AKA attributes, and code, AKA procedures often known as methods.

FP - programming paradigm, style of building the structure
and elements of computer programs. Treates computation as evalutation
of mathematical functions and avoids changing state and mutable data

Two primary components, data (stuff programm knows) and
behaviours (stuff program can do to/with data).

OOP = bring data together and its associated behaviour in
one location (object) to make it easier to understand

FP = data and behaviour distinctively different and should
be seperated for clarity

FP leans heavily on tiny methods that do one small part of a
larger job called composition

FP likes to view everything as a data transformation: you start with
this dataset, apply transformations then generate new dataset

Similarties:

  • Create understandable, flexible programs, bug free.

Preference to use:
If it can be easily described as a result of actions applied to results of other actions, FP is simpler to adopt.
It it can be described as a collection of objects, interacting in a sort relational network, then OOP will work better.

A decent rule of thumb is if you're adding new things OOP tends to be a better fit, if you're adding new operations FP tends to be a better fit.

The general way to think about OOP vs FP, is in terms of "things" vs "actions".

So if you find yourself more so thinking about the problem in terms of "things" doing stuff,
then go with an object approach. Let's say you had a bunch of bouncing balls on the screen,
OOP works well with this because you have ball "things" everywhere and they share properties and functionality.

As far as FP goes, FP is generally more difficult to get but a purely FP approach to a problem
is basically starting out with your data, and then using "actions"/functions to transform it
into some result that you want. So something like getting certain data from a JSON object can be done purely with FP.
